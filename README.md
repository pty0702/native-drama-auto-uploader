# ReCreate AI 微短剧生成与视频号提审工具

ReCreate AI 是一套 Windows 桌面工具，用于把微短剧原始素材批量处理成可提审成品，并自动完成微信视频号剧集上架流程。

## 主要功能

- 批量读取素材文件夹，生成新剧名、简介、海报、成本配置图和转码后的视频。
- 默认使用火山方舟豆包接口，文本模型为 `doubao-seed-2-0-lite-260428`，生图模型为 `doubao-seedream-5-0-260128`。
- 成本配置图采用稳定流程：Word 先填字，再本地贴章，AI 只生成空白木纹桌面背景。
- 上传流程使用 Patchright/Playwright 自动操作微信视频号后台，支持失败重试和跳过失败剧目。
- 客户 API Key 不随交付包打包，首次启动后由用户在界面输入并保存。

## 环境要求

- Windows 10 / Windows 11 64 位
- Microsoft Word 可正常打开和导出 docx
- 手机微信用于扫码登录视频号后台
- 稳定网络，可访问火山方舟 API 和微信视频号后台

## 本地开发

```bash
pip install -r requirements.txt
python -m patchright install chromium
python main.py
```

外部依赖：

- `ffmpeg`
- `ffprobe`
- Microsoft Word 或可用的 `docx2pdf` 转换环境

## 交付包使用

解压 `ReCreate_AI_release_YYYYMMDD_latest.zip` 后，双击：

```text
ReCreate AI/ReCreate AI.exe
```

首次启动会自动生成本机配置文件。用户需要在界面里填写：

- 文本 API Key
- 生图 API Key
- 激活码

交付包里默认只带 `sucai/视频.docx` 和 `sucai/模板.jpg` 两个必要模板，不打包测试 API Key、cookie、上传队列或本机配置。

## 素材要求

每部剧一个素材文件夹，文件夹名作为原始剧名。典型结构：

```text
剧名/
  简介.txt
  第1集.mp4
  第2集.mp4
  ...
  成本表.jpg        # 可选，如果存在则直接使用，不再生成成本配置图
  证明1.jpg         # 可选
  证明2.jpg         # 可选
```

项目根目录的 `sucai` 里需要保留：

- `视频.docx`：成本配置 Word 模板。
- `模板.jpg`：印章素材图，推荐白底，从左到右依次放公章、人名签字、法人章。

不再要求 `照片参考.png`。如果存在，也不会作为必需项。

## 成本配置图生成逻辑

1. 如果素材文件夹内已有 `成本表.jpg`，程序直接使用这张图，跳过成本配置图生成。
2. 如果没有 `成本表.jpg`，程序读取 `sucai/视频.docx`。
3. 在 Word 中填写剧名、集数、总时长。
4. 在 Word 底部日期位置填写当天日期。
5. 调用本机 Word 将 docx 转成高清图片。
6. 从 `sucai/模板.jpg` 中抠出公章、人名签字、法人章。
7. 本地把公章贴到报审机构区域，把人名签字贴到“法定代表人或总编辑”右侧，把法人章贴到人名右侧。
8. 贴人名签字前会检测签名下方黑色文字像素，尽量避开原文重叠。
9. 调用 AI 只生成空白浅色木纹桌面背景，不让 AI 生成文字、纸张或印章。
10. 本地把清晰纸张贴到桌面背景上，输出 3:4 纵向真实拍照感图片。
11. 最后检查分辨率、纸张四角、红章像素和黑字像素，通过后输出 `模版.jpg`。

核心原则：AI 只负责桌面真实感，正文、日期、公司名、签名和印章都由本地确定性处理。

## GUI 注意事项

- 生图 API 地址在界面中隐藏，默认使用火山方舟地址。
- 文本模型、生图模型、制作成本控件禁用鼠标滚轮，避免误滚修改；只能点击或手动输入。
- 上传队列不显示在主页面，上传中可点击“查看上传队列”打开独立窗口。

## 上传逻辑

- 队列状态：`pending` -> `uploading` -> `success` / `failed`。
- 单集上传失败会按页面通用失败提示重试，最多 3 次。
- 重试后仍失败，则判定该剧上传失败，跳过本剧并继续下一部。
- 上传流程结束后，浏览器窗口暂时保留约 1 分钟再关闭，便于观察页面状态。

## 打包

使用项目 `.venv` 打包：

```powershell
.\.venv\Scripts\pyinstaller.exe --noconfirm --clean "ReCreate AI.spec"
```

打包后用 `dist/ReCreate AI` 组装交付目录，并只保留：

- 完整 `_internal` 依赖目录
- `ReCreate AI.exe`
- `sucai/视频.docx`
- `sucai/模板.jpg`

不要打包：

- `db/config.json`
- `cookies/`
- `debug/`
- 测试素材
- API Key 或激活码

## 常见问题

### 生图失败怎么办？

先点击界面里的“测试生图并保存”。如果 API 不可用，程序会直接报错，不再使用本地假图兜底。

### 为什么成本配置图文字比较清楚？

因为文字来自 Word 转图，AI 不负责写字，只负责生成桌面背景。

### 成本图必须放哪些文件？

项目 `sucai` 下必须有 `视频.docx` 和 `模板.jpg`。每部剧素材文件夹内如果有 `成本表.jpg`，则直接使用该图。

### 日志在哪里？

运行日志在界面右侧和上传队列弹窗中展示。调试文件会写入程序目录下的 `debug`。
